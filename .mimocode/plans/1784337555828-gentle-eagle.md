# ExpPoint/AdvPoint 统计脚本

## 背景

用户需要统计游戏数据中所有物品、怪物、Props 的 ExpPoint 和 AdvPoint 属性。脚本独立于 DarkFindV5 项目。

## 方案

在 `/home/mio/fmod/` 创建 `exp_stats.py`，扫描游戏解包数据生成统计报告。

### 数据源

```
/home/mio/fmod/Output/Exports/DungeonCrawler/Content/DungeonCrawler/Data/Generated/V2/
├── Item/Item/          → 物品 (2422 文件, 2284 有 ExpPoint, 1822 有 AdvPoint)
├── Monster/            → 怪物 (3532 文件, 396 有 ExpPoint/AdvPoint)
└── Props/              → Props (977 文件, 159 有 ExpPoint, 145 有 AdvPoint)
```

### 字段名

- `ExpPoint` — 经验值（整数，范围 1-25）
- `AdvPoint` — 点数（整数，范围 1-1000，仅物品和怪物有）

### 脚本功能

1. 递归扫描三种类型目录下的所有 JSON 文件
2. 提取 `Properties.ExpPoint` 和 `Properties.AdvPoint`
3. 终端输出：
   - 总览表（类型/总数/有值数/范围/均值）
   - ExpPoint 分布（按值列出数量 + 柱状图）
   - AdvPoint 分布（同上）
   - TOP 30 排名（按 ExpPoint 降序，再按 AdvPoint 降序）
4. 导出 CSV 文件到 `/home/mio/fmod/exp_stats_output.csv`

### 输出文件

- `/home/mio/fmod/exp_stats.py` — 主脚本
- `/home/mio/fmod/exp_stats_output.csv` — 运行后生成的 CSV

### CSV 列

`type,name,raw_name,rarity,exp_point,adv_point`

### 验证

```bash
cd /home/mio/fmod && python3 exp_stats.py
```

检查终端输出和 CSV 文件内容是否正确。
