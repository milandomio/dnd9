# 修复 TearofHrithurs 缺失翻译

## 原因

游戏数据中存在**拼写不一致**：

| 数据源 | 名称 | 有 'm'？ |
|--------|------|----------|
| Item 定义 (`Id_Item_*.json`) | `TearofHrimthurs` | ✅ |
| Game.json 翻译键 | `Text_DesignData_Item_Item_TearofHrimthurs_5001` | ✅ |
| LootDrop/Spawner 文件名 | `TearofHrithurs` | ❌ |

Spawner `TearofHrithurs` 的 `translation_key` 为空，NameResolver 尝试匹配 `Text_DesignData_Item_Item_TearofHrithurs` 但 Game.json 中不存在该键（实际键名有 'm'），最终返回原始英文名。

## 修复方案

在 `api/src/config.py` 的 `HARDCODED_TRANSLATIONS` 中添加：

```python
"TearofHrithurs": "霜巨人之泪",
```

**为什么不用 ALIAS_MAP：** 即使别名将 `TearofHrithurs` → `TearofHrimthurs`，前缀匹配仍会失败，因为 Game.json 键名是 `TearofHrimthurs_5001`（带 `_5001` 后缀），而前缀扫描不会自动剥离这个后缀。直接硬编码翻译是最简单可靠的方案。

## 修改文件

- `api/src/config.py:57` — 在 `HARDCODED_TRANSLATIONS` 字典中添加一行

## 验证

1. 运行数据管道：`cd api && python main.py`
2. 检查输出 JSON：`cat data/json/items/TearofHrimthurs.json | python3 -m json.tool | grep translation`
3. 检查 coords：`cat data/json/coords/TearofHrithurs.json | python3 -m json.tool | head -20`
